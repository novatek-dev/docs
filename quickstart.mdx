## Get started in 5 minutes

Set up your NovaCloud EIP instance and start processing enterprise data.

### Step 1: Prerequisites

<AccordionGroup>
  <Accordion icon="server" title="System Requirements">
    - Docker & Docker Compose installed
    - Minimum 16GB RAM
    - 50GB free disk space
    - Ubuntu 20.04+ or similar Linux distribution
    
    Check installation:
    ```bash
    docker --version
    docker-compose --version
    ```
  </Accordion>
  
  <Accordion icon="copy" title="Clone Repository">
    ```bash
    git clone https://github.com/novatek-dev/novacloud-eip.git
    cd novacloud-eip
    ```
  </Accordion>
</AccordionGroup>

### Step 2: Configure Environment

<AccordionGroup>
  <Accordion icon="file" title="Setup .env File">
    1. Copy the environment template:
    ```bash
    cp .env.example .env
    ```
    
    2. Update key variables:
    ```env
    NEXT_PUBLIC_API_URL=http://localhost:3000
    DATABASE_URL=postgresql://user:password@postgres:5432/novacloud
    NEO4J_URI=bolt://neo4j:7687
    OLLAMA_BASE_URL=http://ollama:11434
    QDRANT_URL=http://qdrant:6333
    ```
  </Accordion>
  
  <Accordion icon="palette" title="Update Configuration">
    Edit `docker-compose.yml`:
    - Set resource limits per container
    - Configure port mappings
    - Update volume mounts for data persistence
  </Accordion>
</AccordionGroup>

### Step 3: Deploy Containers

<AccordionGroup>
  <Accordion icon="cube" title="Start All Services">
    ```bash
    docker-compose up -d
    ```
    
    Verify all containers are running:
    ```bash
    docker-compose ps
    ```
    
    Expected output:
    - ✅ Container 1: Next.js (Port 3000)
    - ✅ Container 2: Supabase (Port 5432)
    - ✅ Container 3: Ollama (Port 11434)
    - ✅ Container 4: Qdrant (Port 6333)
    - ✅ Container 5: Neo4j (Port 7687)
  </Accordion>
  
  <Accordion icon="check" title="Verify Setup">
    ```bash
    # Check Next.js app
    curl http://localhost:3000
    
    # Check database connection
    docker-compose logs postgres
    
    # Check Ollama models
    curl http://localhost:11434/api/tags
    ```
  </Accordion>
</AccordionGroup>

### Step 4: Import Data

<AccordionGroup>
  <Accordion icon="database" title="Load Initial Snapshot">
    1. Place your SQL snapshot in `./snapshots/` directory
    2. Run the import workflow:
    ```bash
    docker-compose exec postgres psql -U novacloud -f /snapshots/data.sql
    ```
  </Accordion>
  
  <Accordion icon="bolt" title="Initialize LLM Models">
    ```bash
    # Download and cache embedding model
    curl http://localhost:11434/api/pull -d '{"name": "mistral"}'
    curl http://localhost:11434/api/pull -d '{"name": "llama2"}'
    ```
  </Accordion>
</AccordionGroup>

### Step 5: Access Dashboard

Your NovaCloud EIP is ready! Access the interfaces:

| Service | URL | Purpose |
|---------|-----|---------|
| **Web App** | http://localhost:3000 | Main dashboard |
| **Neo4j Browser** | http://localhost:7474 | Graph visualization |
| **Supabase Studio** | http://localhost:8000 | Database management |
| **API Docs** | http://localhost:3000/api/docs | REST API reference |
