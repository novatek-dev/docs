## Page Structure

Every documentation page should follow this format:

1. **Title** - Clear, action-oriented
2. **Description** - What users will learn
3. **Icon** - Relevant emoji for identification
4. **Introduction** - 2-3 sentences explaining content
5. **Main Sections** - Use H2 (##) for sections
6. **Subsections** - Use H3 (###) for details
7. **Code Examples** - Real, runnable code
8. **Tips/Warnings** - Important information
9. **Next Steps** - Link to related pages

## Text Formatting

**Bold** - Important terms and emphasis
`code` - Technical terms, commands, file names
_Italic_ - Alternative names or emphasis

## Container Documentation Format

When documenting a Docker container:

| Field | Example |
|-------|---------|
| **Technology** | Next.js + React |
| **Purpose** | Web application |
| **Port** | 3000 |
| **Environment** | NODE_ENV, NEXT_PUBLIC_API_URL |

## Information Blocks

<Tip>
Docker containers communicate via `container-name:port` within Docker network
</Tip>

<Warning>
Sensitive data like passwords must use environment variables, never hardcode
</Warning>
