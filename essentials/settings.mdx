## ğŸ—ï¸ System Architecture

NovaCloud EIP runs on Novatek EIP Server as a containerized microservices platform with 5 Docker containers coordinating via Docker Network.

### 5-Container Architecture

| Container | Technology | Purpose |
|-----------|-----------|---------|
| **1** | Next.js + Prisma | Web App & Business Logic |
| **2** | Supabase/PostgreSQL | Relational Database & APIs |
| **3** | Ollama + LangChain | AI/LLM & RAG Orchestration |
| **4** | Qdrant | Vector Store for Semantic Search |
| **5** | Neo4j | Graph Database for Relationships |

---

## ğŸŒ Component Details

### Frontend Layer (Container 1)
- **Next.js** - Modern React framework for web UI
- **Prisma ORM** - Type-safe database operations
- **Monitoring API** - Real-time server metrics
- **WebSocket** - Real-time data synchronization

### Database Layer (Container 2)
- **PostgreSQL** - Core relational database
- **PostgREST** - Auto-generated REST APIs
- **GoTrue** - JWT authentication
- **Realtime Server** - WebSocket database changes
- **Storage** - File management
- **Edge Functions** - Serverless compute

### AI/LLM Layer (Container 3)
- **Ollama** - Local LLM inference engine
- **LangChain** - RAG workflow orchestration
- **Models** - Llama 3, Mistral, CodeLlama
- **Embeddings** - Text-to-vector transformation

### Vector Database (Container 4)
- **Qdrant** - High-performance vector store
- **Similarity Search** - Semantic search capabilities
- **RAG Integration** - Retrieval pipeline

### Graph Database (Container 5)
- **Neo4j** - Graph-native database
- **Cypher** - Graph query language
- **Relationships** - Complex entity connections
- **Browser UI** - Graph visualization
